Global Systems & Architecture

The codebase utilizes a modular, event-driven architecture to keep responsibilities separate. Rather than binding all mechanics directly to the player, global state management, scoring, and visuals are distributed across dedicated singletons and managers.

Central Managers

GameManager

File: Assets/_Scripts/Manager/GameManager.cs

The central singleton managing general state (Playing, Victory, GameOver), floor progressions, timers, and upgrades.

  • Manages PlayerLifeTime (depleting every frame) and LevelSurvivalTime.
  • Scales enemy difficulty (HP, speed, cooldowns) as the floor advances.
  • Triggers the Roguelike upgrade selection UI on floor clear.
  • Spawns the level transition portal (Portal.cs) on floor clear.

Score & Combo Managers

Files: ScoreManager.cs & ComboManager.cs

Dedicated scripts separating score accumulation and combo timeout mechanics from the main gameplay loops.

  • ScoreManager aggregates score points upon receiving enemy death events.
  • ComboManager tracks consecutive kills within a 2-second decay window. Under the hood, it triggers screen alerts and relays combo tiers back to the UI.

Event Bus Pattern (GameEvents)

File: Assets/_Scripts/Manager/GameEvents.cs

To prevent tight coupling between managers, the player, and the UI, the game uses static delegates. Below is a breakdown of the active events:

Event Name Parameters Publisher Subscribers / Listeners
OnEnemyDie int score, Vector2 position EnemyController ScoreManager, ComboManager, GameManager
OnPlayerDash None PlayerDashingState GameManager
OnEnemyKilled Vector2 position, int comboCount ComboManager UIComboSpawner
OnScoreAndTimeGained Vector2 pos, int score, float time EnemyController UI Score Tracker / Floating Text (If any)
OnScoreChanged int currentScore ScoreManager InGameHUDController
OnComboChanged int currentCombo ComboManager InGameHUDController
OnSurvivalTimeChanged int secondsRemaining GameManager InGameHUDController

Performance: Object Pooling

Files: ObjectPooler.cs & PoolObject.cs (in Assets/_Scripts/Manager/)

To avoid massive garbage collection overhead and frame drops (lag spikes) due to frequent Instantiate() and Destroy() calls for floating texts and explosion effects, an Object Pool is implemented.

💡 Pool Usage

To spawn a pooled object (e.g. damage indicators), call the singleton method:

ObjectPooler.Instance.Spawn(string key, Vector3 position, Quaternion rotation, float autoReturnTime);

Spawning keys are bound to pool names such as PoolType.DamageText.ToString() or PoolType.DeathVFX.ToString().

DamageText Floating Indicator

File: Assets/_Scripts/Manager/DamageText.cs

  • Automatically assigns itself as a child of the Canvas hierarchy upon initialization.
  • Contains a Setup(string value, Color color) method to dynamically change the text value and color (e.g. Crimson for damage, Cyan for time penalities, Green for time gains).
  • Floats upward at a set pixel speed and automatically returns to the pool after 0.8 seconds by setting gameObject.SetActive(false).

Juice & Aesthetics Setup

Files: CameraJuice.cs & NeonSceneGenerator.cs

  • CameraJuice: Attached to the Main Camera. Uses DOTween to trigger a clean 2D screen shake on high combo kills (Triple Kill and above). Exposes TriggerScreenShake(duration, strength).
  • NeonSceneGenerator: A procedural setup tool for generating the "Neon Noir Minimalist Arcade" arena.
    • Creates boundaries matching the camera aspect ratio with zero-friction, fully bouncy colliders.
    • Procedurally generates a grid of thin neon-colored sprite lines.
    • Automatically sets up URP Volume with custom Bloom (intensity 2.5), Vignette, and Chromatic Aberration.
    • Exposes menu command (Tools/Neon Noir Arcade/Generate Arena) and runs at runtime start.
